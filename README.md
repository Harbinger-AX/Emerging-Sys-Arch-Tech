# Emerging-Sys-Arch-Tech
Repository of files for a class

Summarize the project and what problem it was solving.
  This project aimed to make a functional embedded system by building a circuit centered around a Raspberry Pi 4B containing a temperature and humidity sensor, an LCD display showing the current temp (in F or C), the temp setpoint, and the date and time. Three buttons on the circuit are used to raise or lower the setpoint, and to switch between F and C and show whether the system is actively heating or cooling.

What did you do particularly well?
  I implemented the code well, with clear functions and comments that explain each section's function.

Where could you improve?
  I could have made the physical circut a bit cleaner; making the buttons labeled or spacing them further apart for ergonomics.

What tools and/or resources are you adding to your support network?
  StackOverflow is an invaluable resource, along with GitHub to retrieve common code snippets and frequently used functions or scripts.

What skills from this project will be particularly transferable to other projects and/or course work?
  This project developed my skills for circuit assembly, specifically how to understand GPIO interfaces and common trends and principles with it.

How did you make this project maintainable, readable, and adaptable?
  The comments clearly explain every section of code, and the breadboard circuit has labels on the breakout board along with decent spacing between components to support serviceablility.
